# 灯光设计

`status_led` 的灯光模式、触发条件和播放参数。

## 状态机

灯光状态分为两类：

| 类型 | 行为 | 模式 |
| --- | --- | --- |
| **持续态** | 进入后循环播放，直到设备退出该状态才停止 | 充电、低电量 |
| **瞬态** | 进入后单次播放，播完自动熄灭 | 连接成功、连接断开、模式切换 |


运行时维护两个独立状态：

- `current_persistent`：当前激活的持续态（充电/低电量/无）
- `current_transient`：当前播放中的瞬态 profile 指针（有/无）

规则：

- 持续态互斥：充电和低电量不可能同时存在
- 瞬态可打断持续态：连接/切换事件发生时，中断持续态循环，立即播放瞬态
- 瞬态播完后自动恢复持续态循环
- 瞬态不可被打断：瞬态播放期间产生的持续态变更（如开始充电），等瞬态播完再生效
- 瞬态播放期间再次收到瞬态事件：停止当前瞬态，播新的（互斥）

## 灯光模式

### 1. 连接成功

| 参数 | 值 |
| --- | --- |
| 触发 | 蓝牙/2.4G 连接成功，或 USB 枚举完成 |
| 播放方式 | 单次 |
| 总时长 | ~1100ms |
| 亮度 | 50% |

一次"水滴"效果：300ms 柔和亮起到半亮，800ms 缓缓消散至熄灭。

### 2. 充电中

| 参数 | 值 |
| --- | --- |
| 触发 | 检测到充电接入 |
| 播放方式 | 循环 |
| 单周期 | ~6000ms（渐亮 1500ms + 渐暗 1500ms + 全暗 3000ms） |
| 最大亮度 | 20% |

平缓呼吸，亮的时间短、暗的时间长，尽可能不发光。

### 3. 低电量报警

| 参数 | 值 |
| --- | --- |
| 触发 | 检测到低电压 |
| 播放方式 | 循环 |
| 单周期 | ~5450ms（亮 150ms + 灭 150ms + 亮 150ms + 全暗 5000ms） |
| 亮度 | 40% |

急促双闪后长留白，利用本能警觉，同时极致省电。

### 4. 连接断开

| 参数 | 值 |
| --- | --- |
| 触发 | 蓝牙/2.4G 断开，或 USB 拔出 |
| 播放方式 | 单次 |
| 总时长 | ~300ms |
| 亮度 | 40% |

短闪一次后熄灭，与低电量报警的急促双闪形成区分。

### 5. 模式切换

#### 5a. 切换到蓝牙

| 参数 | 值 |
| --- | --- |
| 触发 | 模式开关拨到蓝牙档 |
| 播放方式 | 单次 |
| 总时长 | ~1200ms |
| 亮度 | 60% |

缓慢闪烁 3 次，每次 200ms 亮 + 200ms 暗。

#### 5b. 切换到 2.4G

| 参数 | 值 |
| --- | --- |
| 触发 | 模式开关拨到 2.4G 档 |
| 播放方式 | 单次 |
| 总时长 | ~900ms |
| 亮度 | 80% |

快速连闪 3 次，每次 150ms 亮 + 150ms 暗。
